# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in FairCom FDW, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, contact the FairCom security team directly:

- **Email:** security@faircom.com
- **Subject:** FairCom FDW Security Vulnerability

Please include:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** Within 2 business days
- **Initial assessment:** Within 5 business days
- **Fix or mitigation:** Depends on severity, typically within 30 days

## Scope

This policy covers the FairCom Foreign Data Wrapper extension for PostgreSQL, including:

- The shared library (`faircom_fdw.so`)
- SQL extension scripts
- Connection handling and credential management
- Data type conversion and query execution

Issues in the FairCom server itself or PostgreSQL should be reported to their respective maintainers.
